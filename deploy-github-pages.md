# Deployment auf GitHub Pages

## Variante A: Neues Repository

1. Auf GitHub ein neues Repository anlegen, z. B. `bus-eignungstraining`.
2. Die Dateien `index.html`, `styles.css`, `app.js` und `README.md` hochladen.
3. In GitHub öffnen: **Settings → Pages**.
4. Bei **Build and deployment** einstellen:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Speichern.
6. Nach kurzer Zeit ist die Seite erreichbar unter:
   `https://DEIN-GITHUB-NAME.github.io/bus-eignungstraining/`

## Variante B: Bestehendes GitHub-Pages-Repository

1. In einem bestehenden öffentlichen Repository einen Unterordner anlegen, z. B. `/bus-eignungstraining/`.
2. Die Dateien in diesen Unterordner legen.
3. Wenn GitHub Pages für das Repository aktiv ist, lautet die URL ungefähr:
   `https://DEIN-GITHUB-NAME.github.io/REPOSITORY-NAME/bus-eignungstraining/`

## Datenschutz-Hinweis

Die Seite enthält bewusst keine Namen, Adressen oder Falldaten. Sie kann öffentlich gehostet werden, ohne Samir namentlich zu nennen.
